# 前后端代码位置说明 — 答辩回答指南

> 当老师问「页面的前后端代码在哪」时，可以用本文档快速定位并口头回答。

---

## 一、30 秒口头回答模板（直接背）

> 老师好，我的项目是**前后端分离**结构，代码分两个部分：
>
> **前端页面**在 `drug-front` 目录下，使用 Vue 3 + Element Plus 开发。具体页面在 `drug-front/src/views/`，每个菜单页面对应一个 `.vue` 文件；路由在 `router/index.js`，接口调用在 `api/` 目录。
>
> **后端接口**在项目根目录的 `src/main/java/com/hospital/drugmanagement/` 下，按分层组织：`controller` 提供 RESTful API，`service` 写业务逻辑，`mapper` 和 `entity` 负责数据库访问。配置文件在 `src/main/resources/application.yml`。
>
> 前后端通过 HTTP 通信：前端请求 `/api/...`，Vite 开发时代理到后端 `localhost:8081`。

## 五、前后端是怎么连起来的（老师常追问）

### 5.1 调用链路（以「药品列表」为例）

```
1. 用户打开 /drug 页面
   → router/index.js 加载 views/drug/DrugList.vue

2. 页面 mounted 时调用 api/drug.js 的 getDrugList()
   → utils/request.js 发 GET /api/drug/list

3. Vite 代理（vite.config.js）
   → 转发到 http://localhost:8081/api/drug/list

4. 后端 DrugInfoController 接收请求
   → DrugInfoServiceImpl 查库
   → 返回 JSON { code, message, data }

5. 前端 request.js 解析响应
   → DrugList.vue 渲染表格
```

### 5.2 关键配置文件

| 文件 | 作用 |
|------|------|
| `drug-front/vite.config.js` | 前端端口 3000，`/api` 代理到 8081 |
| `drug-front/src/utils/request.js` | Axios 基址、自动带 JWT Token |
| `src/main/resources/application.yml` | 后端端口 8081、MySQL 连接、JWT 密钥 |
| `src/main/java/.../config/CorsConfig.java` | 跨域配置 |
| `src/main/java/.../interceptor/AuthInterceptor.java` | 除登录外所有 `/api` 请求需 Token |

### 5.3 口头回答模板

> 前后端通过 RESTful API 通信。前端 Axios 请求 `/api/xxx`，开发环境下 Vite 代理到后端 8081 端口；后端 Controller 返回统一 JSON 格式。登录后 JWT 存在 localStorage，每次请求在 Header 里带 `Authorization: Bearer token`，后端拦截器校验通过才允许访问。

## 七、常见追问与回答

**Q1：为什么前端和后端不在同一个目录？**

> 这是前后端分离的标准做法。前端是 Node/Vue 工程，后端是 Java/Maven 工程，各自独立开发、独立部署。前端打包后是静态文件，后端提供 API 服务。

**Q2：Thymeleaf 是干什么的？你的页面用它了吗？**

> pom.xml 里引入了 Thymeleaf，但本系统页面主要由 Vue 前端渲染，后端以 REST API 为主。Thymeleaf 可作为备用服务端模板，实际业务页面都在 `drug-front/src/views/`。

**Q3：Mapper XML 在哪？**

> 本项目主要使用 MyBatis-Plus 的 BaseMapper 和 Lambda 条件构造器，大部分 CRUD 不需要写 XML。若配置了自定义 SQL，XML 放在 `src/main/resources/mapper/`（当前以注解和 Plus 为主）。

**Q4：前端打包后放哪？**

> 执行 `npm run build` 后生成 `drug-front/dist/`，可部署到 Nginx 等静态服务器；后端单独运行 Spring Boot，两者通过 `/api` 通信。

**Q5：怎么从页面代码找到对应的后端接口？**

> 三步：① 看 `.vue` 文件 import 了哪个 `api/xxx.js`；② 打开 api 文件看请求的 URL；③ 在后端 `controller` 包中找相同路径的 `@RequestMapping`。

**Q6：Entity 和数据库表什么关系？**

> 一一对应。例如 `DrugInfo.java` 对应 `drug_info` 表，MyBatis-Plus 通过 `@TableName` 和字段映射完成 ORM。

## 八、1 分钟完整回答示例（合并版）

> 老师，我这个项目是前后端分离架构。

## 九、IDE 中如何快速打开（答辩演示用）

1. **前端某个页面**：IDE 打开 `drug-front` → `src/views/` → 对应 `.vue`
