## 一、基础数据（主数据）

### 1. `drug_info` — 药品信息表

| 说明 | 内容 |
|---|---|
| **作用** | 存储药品主数据，是系统的「药品字典」 |
| **主要字段** | 药品编码、名称、规格、单位、类型、采购/销售价、预警上下限、供应商、批准文号、上架状态 |
| **关联** | 被库存、入库、出库、采购明细、预警等表引用 |

**典型数据示例：** 阿莫西林胶囊（YP001）、布洛芬片（YP003）

---

### 2. `supplier_info` — 供应商信息表

| 说明 | 内容 |
|---|---|
| **作用** | 管理药品供应商/生产企业信息 |
| **主要字段** | 供应商编码、名称、联系人、电话、地址、营业执照号、启用状态 |
| **关联** | `purchase_order.supplier_id`、`drug_info.supplier_id` |

---

### 3. `warehouse_info` — 仓库信息表

| 说明 | 内容 |
|---|---|
| **作用** | 管理存放药品的仓库或药房 |
| **主要字段** | 仓库编码、名称、地址、管理员、联系电话、启用状态 |
| **关联** | 库存、入库、出库、盘点、预警均按仓库区分 |

**典型数据示例：** 主仓库（WH001）、门诊药房（WH002）、急诊药房（WH003）

---

## 二、采购与审核

### 4. `purchase_order` — 采购订单表

| 说明 | 内容 |
|---|---|
| **作用** | 采购单主表，记录一次采购申请的整体信息 |
| **主要字段** | 订单编号、供应商 ID、订单日期、总金额、状态、备注、创建人 |
| **状态流转** | 0 待审核 → 1 已审核 → 2 已入库 / 3 已取消 |

---

### 5. `purchase_order_item` — 采购订单明细表

| 说明 | 内容 |
|---|---|
| **作用** | 一张采购单中具体采购的每种药品 |
| **主要字段** | 订单 ID、药品 ID、采购数量、采购单价、小计金额 |
| **关系** | 多对一关联 `purchase_order`（一张单有多条明细） |

---

### 6. `audit_record` — 审核记录表

| 说明 | 内容 |
|---|---|
| **作用** | 记录采购单的审批历史 |
| **主要字段** | 采购单 ID、审核人、审核结果（1 通过 / 2 驳回）、审核意见、审核时间、审核级别 |
| **关系** | 多对一关联 `purchase_order` |

**业务流程：** 采购员建单 → 审核员审批 → 库管入库 → 采购单状态更新

---

## 三、库存核心

### 7. `drug_stock` — 药品库存表 ⭐

| 说明 | 内容 |
|---|---|
| **作用** | **当前实时库存**，系统最核心的库存数据表 |
| **主要字段** | 药品 ID、仓库 ID、批号、库存数量、锁定数量、生产日期、效期 |
| **唯一约束** | 同一「药品 + 仓库 + 批号」只有一条记录 |
| **更新方式** | 入库增加、出库减少、盘点调整、锁定改变 lock_num |

> 可用库存 = `quantity` - `lock_num`

---

### 8. `drug_in` — 药品入库表

| 说明 | 内容 |
|---|---|
| **作用** | 记录每次药品入库的流水 |
| **主要字段** | 入库单号、采购单 ID、入库类型、药品、仓库、批号、数量、单价、生产日期/效期、操作人 |
| **入库类型** | 采购入库、补货入库、退货入库、调拨入库等 |
| **关联** | 入库成功后更新 `drug_stock` |

---

### 9. `drug_out` — 药品出库表

| 说明 | 内容 |
|---|---|
| **作用** | 记录每次药品出库的流水 |
| **主要字段** | 出库单号、药品、仓库、批号、数量、单价、出库类型、操作人 |
| **出库类型** | 科室领用、销售、报损等 |
| **关联** | 出库成功后扣减 `drug_stock` |

---

### 10. `drug_lock` — 药品锁定记录表

| 说明 | 内容 |
|---|---|
| **作用** | 出库前锁定指定批次库存，防止超卖 |
| **主要字段** | 锁定单号、药品、仓库、批号、锁定数量、锁定原因、锁定/解锁人、状态 |
| **状态** | 0 锁定中 / 1 已解锁 / 2 已取消 |
| **关联** | 锁定数量同步到 `drug_stock.lock_num` |

---

## 四、盘点

### 11. `stock_check` — 库存盘点表

| 说明 | 内容 |
|---|---|
| **作用** | 盘点单主表，发起一次库存盘点任务 |
| **主要字段** | 盘点单号、仓库 ID、盘点日期、状态、盘点人、备注 |
| **状态** | 0 盘点中 / 1 已完成 / 2 已取消 |

---

### 12. `stock_check_item` — 库存盘点明细表

| 说明 | 内容 |
|---|---|
| **作用** | 每个药品批次的盘点明细 |
| **主要字段** | 盘点单 ID、药品 ID、批号、系统数量、实盘数量、差异数量、处理方式 |
| **关系** | 多对一关联 `stock_check` |
| **完成盘点** | 根据差异回写 `drug_stock` 数量 |

**示例：** 系统 78、实盘 75 → 差异 -3 → 库存调整为 75

---

## 五、预警

### 13. `stock_warning` — 库存预警记录表

| 说明 | 内容 |
|---|---|
| **作用** | 存储系统自动或手动产生的各类库存预警 |
| **主要字段** | 预警单号、药品、仓库、批号、当前库存、预警类型、预警级别、处理状态、处理人、处理建议 |
| **预警类型** | 0 低库存 / 1 积压 / 2 临期 / 3 过期 / 4 滞销 |
| **处理状态** | 0 未处理 / 1 已处理 |

**触发时机：** 出入库后自动检测、手动扫描临期/滞销、修改预警阈值时

## 七、表关系概览

```
基础数据
├── drug_info          药品字典
├── supplier_info      供应商
└── warehouse_info     仓库/药房

采购链路
purchase_order ──→ purchase_order_item   （一对多：主单-明细）
       │
       ├──→ audit_record                  （一对多：审批记录）
       └──→ drug_in                        （入库关联采购单）

库存链路
drug_in  ──→ drug_stock  ←── drug_out     （流水改库存）
drug_lock ──→ drug_stock                   （锁定改 lock_num）
stock_check ──→ stock_check_item ──→ drug_stock  （盘点调整）
drug_stock ──→ stock_warning               （触发预警）

权限体系
sys_user ──→ sys_role ──→ sys_role_menu ←── sys_menu
sys_operation_log                          （操作审计）
system_notice                              （系统公告）
```

## 八、答辩速记

| 分类 | 包含表 | 一句话 |
|---|---|---|
| **主数据** | drug_info、supplier_info、warehouse_info | 药品、供应商、仓库的基础档案 |
| **采购** | purchase_order、purchase_order_item、audit_record | 买药的申请、明细和审批 |
| **库存流水** | drug_in、drug_out | 每次进出的历史记录 |
| **库存现状** | drug_stock | 当前每个批次还剩多少（最核心） |
| **库存管控** | drug_lock、stock_check、stock_check_item、stock_warning | 锁定、盘点、预警 |
| **系统支撑** | sys_user、sys_role、sys_menu、sys_role_menu、sys_operation_log、system_notice | 用户、权限、日志、公告 |
