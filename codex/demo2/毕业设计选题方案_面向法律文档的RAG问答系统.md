# 毕业设计选题方案

## 题目

**面向法律文档的层级化检索增强生成系统：结合语义分块与多粒度重排序**

---

## 一、选题背景与意义

### 1.1 现实需求

法律领域拥有海量的非结构化文档（法规条文、裁判文书、合同模板、司法解释等）。法律从业者在进行案情分析、法条检索、类案比对时，依赖大量人工阅读与经验判断，效率瓶颈明显。大语言模型虽具备强大的文本生成能力，但在专业法律场景中面临两个突出问题：

- **知识时效性**：模型训练数据截止于某个时间点，无法覆盖最新颁布或修订的法律法规。
- **幻觉问题**：模型可能生成看似合理但于法无据的内容，在法律场景中不可接受。

检索增强生成（Retrieval-Augmented Generation, RAG）通过引入外部知识库来缓解上述问题，已成为业界主流方案。然而，法律文档具有**篇幅长、结构复杂、术语密集、逻辑层级深**等特点，通用 RAG 方案直接套用往往效果不佳。

### 1.2 研究意义

- **理论意义**：探索面向长文本、层级化结构的文档分块策略与检索排序优化方法，丰富 RAG 在垂直领域的技术体系。
- **应用价值**：构建一套可部署的法律文档智能问答原型系统，辅助法律从业者提升信息检索与文书撰写效率。

---

## 二、国内外研究现状

| 方向 | 代表性工作 | 不足 |
|------|-----------|------|
| 通用 RAG 框架 | LangChain、LlamaIndex、Haystack | 分块策略简单（固定 token 切片），未利用文档结构 |
| 法律 NLP | Lawformer、Legal-BERT、ChatLaw | 侧重语义理解，缺乏与外部知识库的检索-生成联动 |
| 检索排序优化 | ColBERT-v2、Cohere Rerank、BGE-Reranker | 多粒度融合不足，未针对法律层级结构设计 |
| 文档分块 | Semantic Chunking、Late Chunking | 计算开销大，对长文档的边界划分不够鲁棒 |

现有方法的主要空白：**缺乏一种能够感知法律文档层级结构（章-节-条-款-项）的分块与检索方案，且端到端的检索-生成联合优化尚未被充分探索。**

---

## 三、研究目标与内容

### 3.1 总体目标

设计并实现一套面向法律文档的层级化 RAG 问答系统，在检索精度与生成质量上均显著优于通用 RAG 基线，并在公开法律 QA 数据集上达到领先水平。

### 3.2 具体研究内容

**内容一：法律文档层级感知分块算法**

- 利用正则解析与轻量级分类器自动识别法律文档的层级边界（章、节、条、款、项）。
- 设计**混合分块策略**：以"条"为基本检索单元，同时保留上下文层级信息作为辅助信号。
- 对比固定 token 分块、递归分块、语义分块等基线方法。

**内容二：多粒度检索与层级感知重排序**

- 第一阶段（召回）：使用稠密检索（如 bge-large-en-v1.5）与稀疏检索（BM25）的混合召回。
- 第二阶段（重排序）：设计层级感知重排序模型，将文档的层级距离、父子关系编码为排序特征，提升上下文相关段落优先级的排序准确性。
- 引入查询改写模块，对法律术语进行同义扩展与缩写还原。

**内容三：检索增强的生成模块与幻觉抑制**

- 基于开源 LLM（如 Qwen2.5-7B-Instruct 或 ChatGLM3-6B）进行法律领域指令微调。
- 在生成阶段引入**引用标注机制**：模型输出时自动标注所引用的法条编号，便于用户溯源验证。
- 设计基于 entailment 的后处理校验模块，对生成内容与检索片段进行一致性检查，降低幻觉。

**内容四：系统实现与评测**

- 构建完整的 RAG 流水线系统，提供 RESTful API 与简易 Web 交互界面。
- 在法律问答数据集（如 CAIL 2019 阅读理解子集、LeCaRD 类案检索数据集）上进行评测。

---

## 四、技术路线

```
法律文档库 -> 层级感知分块 -> 向量索引 + BM25 索引
                                    |
用户问题 -> 查询改写 -> 混合召回 -> 层级感知重排序 -> LLM 生成 -> 引用校验 -> 最终答案
```

### 4.1 分块模块

- 输入：原始法律文档（PDF/TXT）
- 处理：正则匹配 + Layout 解析器识别层级结构
- 输出：带层级标签的文本块列表，每块保留 {level, parent_id, sibling_order} 元数据

### 4.2 检索模块

- 嵌入模型：BAAI/bge-large-en-v1.5（或 BAAI/bge-m3，支持多语言）
- 检索方式：向量相似度（余弦）+ BM25 加权融合
- 重排序模型：BAAI/bge-reranker-v2-m3，额外注入层级特征

### 4.3 生成模块

- 基座模型：Qwen2.5-7B-Instruct
- 微调方式：LoRA（rank=16, alpha=32）
- 训练数据：法律问答对 + 检索片段拼接

### 4.4 幻觉抑制

- 引用标注：在生成文本中以 [法条编号] 格式标注来源
- 一致性校验：使用 NLI 模型判断生成内容是否被检索片段蕴含

---

## 五、创新点

1. **层级感知分块策略**：区别于固定 token 切片，利用法律文档的结构化特征进行语义完整的分块，同时保留上下文层级信息。

2. **多粒度层级重排序**：在通用重排序模型基础上引入文档层级距离与父子关系特征，提升相关段落排序的上下文准确性。

3. **端到端引用溯源机制**：生成阶段自动标注法条来源，配合 NLI 校验，在法律场景中实现可解释、可验证的问答输出。

## 六、实验方案

### 6.1 数据集

| 数据集 | 用途 | 规模 |
|--------|------|------|
| CAIL 2019 阅读理解 | 端到端问答评测 | ~3,000 问答对 |
| LeCaRD（类案检索） | 检索阶段评测 | ~10,000 查询-文档对 |
| 自建法律文档库 | 构建检索索引 | ~50,000 篇裁判文书/法规 |

### 6.2 基线方法

- **Naive RAG**：固定 512 token 分块 + 向量检索 + LLM 生成
- **LangChain Recursive RAG**：递归字符分块 + 向量检索
- **LlamaIndex Tree RAG**：树状摘要索引 + 检索
- **ChatLaw**：法律专用 LLM（无检索增强）

### 6.3 评价指标

| 阶段 | 指标 |
|------|------|
| 检索 | Recall@k、MRR、NDCG@k |
| 生成 | BLEU、ROUGE-L、F1 |
| 综合 | 答案准确率、引用准确率、人工评分（1-5） |

### 6.4 消融实验

- 层级感知分块 vs. 固定分块
- 混合召回 vs. 单一召回
- 层级重排序 vs. 通用重排序
- 引用校验模块有无
