
# 985 高校计算机硕士毕业设计：选题方向与要求

## 一、总体定位

985 高校计算机科学与技术（或软件工程）硕士毕业设计，要求在导师指导下独立完成一项具有**理论深度**或**工程创新性**的科研或开发工作。成果应体现：

- 对相关领域前沿或经典问题的系统理解
- 独立的问题分析、方案设计与实现能力
- 实验或验证环节的严谨性
- 学术写作的规范性

最终交付物包括：毕业设计论文（通常 3–5 万字）、可运行的代码/系统/数据集、以及答辩 PPT。

---

## 二、选题基本原则

1. **专业相关性** — 选题须属于计算机科学与技术一级学科范畴，包括但不限于人工智能、计算机视觉、自然语言处理、数据挖掘、计算机系统、软件工程、网络与安全、人机交互、计算机图形学等方向。

2. **创新性** — 在方法、应用场景或系统架构上有所改进或突破，避免简单的复现或拼凑。

3. **可行性** — 在一年左右的时间内，具备可获取的数据集、算力资源或实验环境，技术路线清晰。

4. **工作量饱满** — 应有足够的算法设计、系统实现、实验对比与结果分析内容，通常要求核心代码量不低于 5000 行（不含第三方库）。

5. **可验证** — 研究结果可通过公开数据集评测、用户实验、对比基线或理论证明等方式得到客观验证。

---

## 三、常见选题方向与示例题目

### 3.1 人工智能 / 深度学习

- **面向小样本场景的图神经网络节点分类方法研究**
  - 核心：结合元学习或对比学习，提升 GNN 在标签稀疏场景下的泛化能力。
  - 验证：在 Cora、Citeseer、PubMed 等基准上对比 GCN、GAT、GraphSAGE 等基线。

- **基于扩散模型的可控图像生成与编辑方法**
  - 核心：研究文本引导下的局部编辑与风格迁移，改进采样效率或编辑精度。
  - 验证：在 FFHQ、ImageNet 等数据集上做定量（FID、CLIP score）与定性评估。

- **面向时序数据的 Transformer 架构优化**
  - 核心：改进自注意力机制在长序列上的计算效率，或引入可解释性模块。
  - 验证：在电力负荷、交通流量、金融时序等公开数据集上对比 LSTM、Informer、TimesNet。

### 3.2 计算机视觉

- **跨模态行人重识别中的模态对齐与特征融合**
  - 核心：解决可见光-红外跨模态检索中的模态差异问题，设计特征对齐模块。
  - 验证：在 SYSU-MM01、RegDB 数据集上评测 Rank-1、mAP。

- **面向自动驾驶的实时语义分割轻量化网络设计**
  - 核心：在保持分割精度的前提下，通过剪枝、蒸馏或高效卷积设计降低计算量。
  - 验证：在 Cityscapes、CamVid 上对比 mIoU 与 FPS。

- **基于隐式神经表示的三维场景重建**
  - 核心：利用 NeRF 或其变体，从稀疏视角图像重建连续三维场景。
  - 验证：在 DTU、BlendedMVS 上对比 PSNR、SSIM、Chamfer Distance。

### 3.3 自然语言处理

- **面向垂直领域的大语言模型指令微调与评测**
  - 核心：在医疗/法律/金融等垂直领域构建指令数据集，对开源基座模型（如 Llama、Qwen、ChatGLM）进行 LoRA 微调。
  - 验证：设计领域评测集，对比微调前后在自动指标与人工评估上的表现。

- **基于检索增强生成（RAG）的知识密集型问答系统**
  - 核心：融合稠密检索与 LLM 生成，研究分块策略、重排序与答案合成的优化。
  - 验证：在 Natural Questions、HotpotQA 等数据集上评测 F1、EM。

- **代码生成中的语法约束解码方法**
  - 核心：在 LLM 解码过程中引入类型系统或语法约束，保证生成代码的可编译性。
  - 验证：在 HumanEval、MBPP 上对比 pass@k 与编译通过率。

### 3.4 数据挖掘 / 推荐系统

- **基于图神经网络的会话推荐系统**
  - 核心：利用 session 级图建模用户行为序列，融合长期偏好与短期意图。
  - 验证：在 Yoochoose、Diginetica 上对比 Recall、MRR。

- **多兴趣解耦的序列推荐模型**
  - 核心：通过胶囊网络或注意力路由将用户历史行为分解为多个兴趣表征。
  - 验证：在 Amazon、MovieLens 等数据集上对比 HR、NDCG。

### 3.5 计算机系统 / 软件工程

- **面向微服务架构的智能根因定位系统**
  - 核心：基于调用链追踪与指标异常检测，构建故障传播图并定位根因服务。
  - 验证：在公开微服务基准（如 TrainTicket、SockShop）上评估准确率与定位时间。

- **基于 eBPF 的容器网络性能监控与诊断工具**
  - 核心：利用 eBPF 技术在内核态采集网络包与系统调用，实现低开销的实时监控。
  - 验证：对比传统工具（如 tcpdump、iperf）的性能开销与监控精度。

### 3.6 网络与信息安全

- **面向物联网固件的二进制漏洞自动化挖掘**
  - 核心：结合符号执行与模糊测试，提升 IoT 固件中漏洞发现的覆盖率。
  - 验证：在公开固件数据集上统计漏洞发现数量与误报率。

- **联邦学习中的隐私攻击与防御机制研究**
  - 核心：实现梯度泄露攻击（如 DLG、iDLG），并设计差分隐私或安全聚合防御方案。
  - 验证：在图像分类任务上评估攻击成功率与模型精度之间的 trade-off。

### 3.7 人机交互 / 计算机图形学

- **基于手部姿态估计的虚拟交互系统**
  - 核心：利用 MediaPipe 或深度学习模型实现实时手部关键点检测，驱动虚拟场景交互。
  - 验证：用户实验评估交互延迟、完成率与用户体验评分。

- **面向数字人的面部动画生成方法**
  - 核心：从音频驱动三维面部网格动画，研究语音-表情映射模型。
  - 验证：在 VOCASET 等数据集上对比唇形同步精度与主观自然度评分。

## 四、毕业设计流程与时间节点（建议）

| 阶段 | 时间 | 主要任务 |
|------|------|----------|
| 选题与开题 | 第 1–2 个月 | 阅读文献，确定选题，撰写开题报告，完成开题答辩 |
| 方案设计与数据准备 | 第 3–4 个月 | 完成技术方案设计，收集/标注数据集，搭建实验环境 |
| 核心算法/系统实现 | 第 5–7 个月 | 完成核心代码开发，进行初步实验 |
| 实验与调优 | 第 8–9 个月 | 完成全部实验，进行对比分析、消融实验、参数调优 |
| 论文撰写 | 第 10–11 个月 | 撰写毕业论文，修改完善 |
| 查重、盲审与答辩 | 第 12 个月 | 提交论文查重，送审，准备答辩 PPT，完成答辩 |

## 五、论文撰写要求

1. **结构完整** — 一般包括：摘要（中英文）、目录、符号说明（可选）、引言、相关工作、方法设计、实验与分析、总结与展望、参考文献、附录。

2. **学术规范** — 引用格式统一（推荐 GB/T 7714 或 IEEE 格式），严禁抄袭，查重率通常要求低于 10%–15%（视学校具体规定）。

3. **图表清晰** — 实验数据以表格或图表呈现，图表须有编号与标题，坐标轴标注清晰。

4. **代码可复现** — 论文附录或附带的代码仓库应包含完整的运行说明、依赖环境与数据预处理脚本。

---

## 六、答辩常见考察点

- 选题背景与意义是否清晰
- 相关工作调研是否充分
- 方法创新点是否明确，与基线相比的优势是否有数据支撑
- 实验设计是否合理，结果分析是否深入
- 对方法的局限性是否有客观认识
- 回答问题是否准确、逻辑清晰

---

## 七、注意事项

- 选题应尽早与导师沟通确认，避免中途大方向调整。
- 保持实验记录习惯，包括参数配置、实验结果、失败尝试等，有助于论文撰写与复盘。
- 论文写作不要等到实验全部完成再开始，建议边实验边写方法章节与相关工作章节。
- 代码与数据注意备份，推荐使用 Git 进行版本管理。
- 关注学校对论文格式（模板）的具体要求，提前使用模板撰写。
