## 一、Cadence 软件系列硬件需求

Cadence 旗下工具覆盖 PCB 设计、模拟IC设计、数字IC设计、物理验证、仿真等多个环节，**不同工具对硬件资源的偏好差异很大**，需要区分看待。

### 1.1 PCB 设计类（Allegro / OrCAD）

> 适合：板级设计、原理图、Layout、信号完整性分析

| 配置项 | 最低要求 | 推荐配置 | 说明 |
|--------|---------|---------|------|
| CPU | Intel Core i7 4.3GHz / AMD Ryzen 7，4核+ | i9 / Ryzen 9，8核+ | 吃单核主频，多核次要 |
| 内存 | 16GB | 32~64GB | 大型 PCB 吃内存（多层、高密度） |
| GPU | 2GB VRAM，支持 OpenGL | RTX A2000~A4000，8~16GB | **必须专业卡**，消费卡在 3D 视图下卡顿严重 |
| 存储 | 50GB SSD | 1TB NVMe SSD | 库文件和项目文件占用大 |
| 显示 | 1920×1200，真彩色 | **双屏** 4K | 物理设计强烈推荐双屏 |
| 操作系统 | Windows 10/11 64bit Pro / Linux | 同左 | 不支持 Home 版 |

> **关键点**：Allegro 的 3D Canvas 和 OpenGL 渲染对专业显卡比较敏感，实测 RTX 3000 Ada 专业卡比 RTX 4080 消费卡渲染帧率高 2~3 倍（驱动差异）。

### 1.2 模拟/混合信号设计类（Virtuoso）

> 适合：模拟 IC 设计、原理图输入、版图编辑、混合信号仿真

| 配置项 | 推荐配置 | 说明 |
|--------|---------|------|
| CPU | **单核性能优先**，高主频 i9 / Ultra 9 | SPICE 仿真正在单核上跑，核多了没用 |
| 内存 | 64~128GB DDR5 ECC | 加载大型版图和仿真波形 |
| GPU | RTX A4000 16GB 或更高 | OpenGL 加速版图显示 |
| 存储 | 2TB NVMe + 4TB SSD | 项目数据和仿真结果占用大 |
| 网络 | **低延迟优先** | 远程 X11 显示场景下延迟 ≤50ms |

> **关键点**：Virtuoso 最大特点是**远程 X11 显示**——用户可能在工作站上操作，实际计算在服务器上跑，此时网络延迟直接影响鼠标交互体验。

### 1.3 数字实现类（Genus / Innovus / Tempus）

> 适合：逻辑综合、布局布线、静态时序分析

| 配置项 | 入门级 | 专业级 | 旗舰级 |
|--------|-------|-------|-------|
| CPU | 16 核 | 32~64 核 | 128 核+ |
| 内存 | 64GB | 128~256GB | 512GB~1TB |
| 存储 | 1TB NVMe | 4TB NVMe | 4TB+ 全闪存 |
| GPU | 不需要 | 一般不需要 | 某些场景需要 |
| 典型机型 | 单路工作站 | 双路 Xeon / EPYC | 双路旗舰服务器 |

> **关键点**：Innovus 布局布线对内存和核心数都有良好扩展性，是典型的"堆核就快"。Tempus 时序分析也支持 64 核+并行。

### 1.4 物理验证类（PVS / Pegasus）

> 适合：DRC、LVS 检查

| 配置项 | 推荐配置 | 说明 |
|--------|---------|------|
| CPU | **核心越多越好**，64~512核 | 并行性极好，可集群扩展 |
| 内存 | 512GB~1TB | 全芯片的规则检查和图形数据占用极大 |
| 存储 | NVMe SSD 4TB+ | 读写 I/O 频繁 |
| GPU | 基本不需要 | 纯计算密集 |
| 网络 | 25GbE 起，推荐 100GbE | **多机分布时对网络带宽要求最高** |

> **关键点**：物理验证是所有 EDA 环节里对硬件最吃"量"的，内存不足直接崩溃，核心不够跑几天。适合配置大内存+多核心的服务器集群。

### 1.5 仿真类（Xcelium / Spectre X）

> 适合：RTL 仿真、模拟电路仿真、UVM 验证

| 配置项 | Xcelium（逻辑仿真） | Spectre X（模拟仿真） |
|--------|-------------------|---------------------|
| CPU | 64~128核，多核并行好 | 单核性能优先，但支持 MPI 分布式 |
| 内存 | 64~256GB | 64~128GB |
| GPU | 支持 Palladium 加速 | **支持 GPU 加速**（需 NVIDIA A100/RTX 6000） |
| 存储 | 高 IOPS（波形读写） | 高 IOPS |
| 网络 | 万兆起 | 万兆起（分布式模式下需要） |

> **关键点**：Spectre X 是 Cadence 中支持 GPU 加速最成熟的工具，如果客户大量做模拟仿真，配 GPU 会有明显收益。

## 二、华大九天软件系列硬件需求

华大九天是国产 EDA 龙头，主要覆盖**模拟设计全流程**（对标 Cadence Virtuoso）和**版图验证**等环节。

### 2.1 模拟设计全流程（Aether / ALPS）

> 适合：模拟电路原理图设计、SPICE 仿真、版图编辑、物理验证

| 配置项 | 入门配置 | 推荐配置 | 旗舰配置 |
|--------|---------|---------|---------|
| CPU | Core i7 / Ultra 7，高主频 | Core Ultra 9 / Xeon W | Threadripper PRO 7995WX |
| 内存 | 32GB | 64~128GB DDR5 ECC | 256GB DDR5 ECC |
| GPU | RTX 4060 8GB | RTX A4000 16GB | RTX A6000 48GB |
| 存储 | 1TB NVMe | 2TB+4TB NVMe | 4TB NVMe Gen5 |
| 操作系统 | Windows 10/11 / Linux | Linux（Ubuntu/RHEL） | Linux |

> **关键点**：华大九天的模拟 EDA 和 Virtuoso 逻辑类似——**吃单核性能、吃内存容量**，不需要太多核，但主频和 IPC 要够高。如果涉及射频仿真（RF），内存需求会更高。

### 2.2 数字设计与验证

> 适合：逻辑综合、仿真验证、形式验证

| 配置项 | 推荐配置 | 说明 |
|--------|---------|------|
| CPU | 双路 Xeon / EPYC，64核+ | 支持多核并行 |
| 内存 | 256~512GB | 支撑大规模逻辑仿真 |
| GPU | 可选（视具体工具） | 部分验证工具可加速 |
| 存储 | 4TB NVMe + 大容量数据盘 | 海量仿真结果 |
| 操作系统 | Linux（推荐） | |

### 2.3 版图物理验证（DRC / LVS）

> 适合：芯片物理验证、规则检查

| 配置项 | 推荐配置 | 说明 |
|--------|---------|------|
| CPU | 双路 EPYC/Xeon，64核+ | 可扩展至 512核+ |
| 内存 | 512GB~1TB | 版图数据 + 规则数据库 |
| 存储 | NVMe SSD + 大容量 RAID | 高 IOPS |
| 网络 | 25GbE+（多机场景） | 分布式验证必要 |

## 三、多机组网网络带宽要求

这是客户最关心的核心问题之一。EDA 工具的多机运行模式分为几种，对网络需求差异很大。

### 3.1 单机工作站场景

如果每台机器各自独立运行（一人一台工作站），网络只用于：

| 用途 | 要求 | 说明 |
|------|------|------|
| License 服务器访问 | 千兆够用 | 仅在启动时校验，流量极小 |
| 文件共享（NFS/Samba）| 千兆够用 | 如果文件不大，千兆即可 |
| 版本管理（Git/SVN）| 千兆够用 | 偶尔提交代码 |

> **结论：单机场景下千兆以太网完全够用。**

### 3.2 多机网络互联场景

当使用**分布式/并行模式**时（多台机器共同跑一个任务），网络成为瓶颈。

| 应用场景 | 涉及工具 | 带宽要求 | 延迟要求 |
|---------|---------|---------|---------|
| **分布式物理验证** | PVS/Pegasus MTFlex | **25GbE 起，推荐 100GbE** | ≤1ms |
| **分布式时序分析** | Tempus | 10~25GbE | ≤5ms |
| **分布式仿真** | Xcelium、Spectre X XDP | 10~25GbE | ≤5ms |
| **布局布线分布式** | Innovus MPI | **25~100GbE** | ≤1ms |
| **远程 X11 显示** | Virtuoso 远程操作 | 千兆~万兆 | **≤50ms（推荐 <10ms）** |
| **NFS 共享存储访问** | 所有工具 | 10~25GbE | ≤50ms |
| **华大九天分布式验证** | 版图验证 | **10~25GbE** | ≤5ms |

### 3.3 网络架构建议

**方案 A：入门级（3~5台，轻量级分布式）**
```
各工作站 ── 万兆交换机 ── 存储服务器
                      │
                  License 服务器（千兆即可）
```
- 网络：万兆以太网（10GbE）
- 交换机：1~2台万兆交换机
- 线缆：CAT6A 或 SFP+ 光缆
- 适用：小团队，偶尔跑分布式验证

**方案 B：专业级（5~20台，中度分布式）**
```
计算节点 ── 25GbE ── 核心交换机（带 RDMA）
          │
    全闪存 NAS/SAN ── 25GbE
          │
    管理网络 1GbE（IPMI）
```
- 网络：25GbE RoCE v2（支持 RDMA）
- 交换机：25GbE 交换机 + 管理交换机
- 适用：中型团队，常用分布式物理验证

**方案 C：旗舰级（20+台，重度分布式）**
```
计算节点 ── InfiniBand NDR 200/400Gb ── IB 交换机
          │
    并行文件系统（Lustre/GPFS）
          │
    管理网络 25GbE
          │
    License 服务器（千兆）
```
- 网络：InfiniBand NDR 200/400Gb 或 100GbE RoCE
- 交换机：InfiniBand 交换机
- 适用：大型团队，全芯片物理验证/大规模分布式仿真

### 3.4 共享存储需求

多台机器通过网络共享设计数据时：

| 项目 | 入门 | 专业 | 旗舰 |
|------|------|------|------|
| 协议 | NFS v4 | NFS + 并行文件系统 | Lustre / GPFS / BeeGFS |
| 带宽 | 10GbE | 25GbE | 100GbE+ |
| 容量 | 10~50TB | 50~200TB | 200TB+ |
| 介质 | SATA SSD | NVMe SSD | 全闪存阵列 |
| 备注 | 单机使用时可能不需共享存储 | 建议加一台 NAS | 需要专业存储团队维护 |

## 四、平台选型建议

### 4.1 按设计阶段推荐配置

| 角色 | 推荐平台 | CPU | 内存 | GPU | 网络 |
|------|---------|-----|------|-----|------|
| PCB 设计工程师 | 工作站 | i9 / Ryzen 9，8核 | 32~64GB | RTX A2000/A4000 | 千兆 |
| 模拟 IC 设计 | 工作站 | Ultra 9 / Xeon W | 64~128GB | RTX A4000+ | 千兆~万兆 |
| 数字后端工程师 | 服务器 | 双路 EPYC/Xeon，64核 | 256GB+ | 可选 | 25GbE |
| 物理验证工程师 | 服务器集群 | 128~512核/集群 | 512GB~1TB | 不需要 | 25~100GbE |
| 仿真工程师 | 服务器 | 64~128核 | 128~256GB | A100（Spectre X） | 10~25GbE |

### 4.2 服务器 vs 工作站对比

| 维度 | 工作站 | 服务器 |
|------|--------|--------|
| 适用场景 | 前端设计（PCB/模拟） | 后端计算（验证/仿真/布�线） |
| GPU 支持 | 好，单卡或双卡 | 可多卡，但散热受限 |
| 内存扩展 | 128~256GB | 1~4TB |
| 多核 CPU | 单路，最多 96核 | 双路，最多 256核+ |
| 噪音 | 一般 | 较大（需机房） |
| 显示输出 | 有 | 通常无（IPMI 管理） |
| 典型部署 | 工程师桌面 | 机房集中管理 |

**建议策略**：
- **前端/交互类**（Virtuoso、Allegro、Aether）：用工作站，放在工程师桌面
- **后端/计算类**（PVS、Innovus、Tempus、Spectre X）：用服务器，集中放在机房，通过网络调度

### 4.3 关键配置原则

1. **内存优先，其次核心数** — EDA 工具内存不够直接崩溃，CPU 慢一点只是等
2. **物理验证是资源大户** — 如果要跑 PVS/Pegasus 级 DRC/LVS，做好 512GB+ 内存+分布式集群的准备
3. **专业显卡不可替代** — 消费卡在 EDA 专业软件下兼容性和性能都不行，尤其 Virtuoso 和 Allegro
4. **网络接口卡选型** — 如果要用分布式计算，网卡必须原厂（Mellanox/Intel），不建议用板载廉价网卡
5. **操作系统** — 大部分 Cadence/华大九天 Linux 工具在 RHEL 8/9 上表现最佳，Ubuntu 也可能但需要确认兼容性矩阵
6. **License 服务器** — 单独一台低配机器即可（2核+4GB），但必须稳定，网络要通
